# Functional Requirements (The "What")
*Syntax: EARS (Easy Approach to Requirements Syntax)*

## 1. Ubiquitous Requirements (Always True)
*   **REQ-001:** The system shall ...
*   **REQ-002:** The system shall ...

## 2. Event-Driven Requirements (When... Then...)
*   **REQ-003:** When [Trigger], the system shall [Response].
*   **REQ-004:** When [Trigger], the system shall [Response].

## 3. State-Driven Requirements (While... Then...)
*   **REQ-005:** While [State], the system shall [Behavior].

## 4. Unwanted Behavior (If... Then... NOT)
*   **REQ-006:** If [Condition], the system shall NOT [Bad Outcome].
